# Interface FAQ

## Q: What's the difference between Console and Tasks? Which should I use?

**A**: Console is a dedicated environment for Recipe development, while Tasks is for deterministic task execution. Both auto-approve all operations.

| Feature | Console | Tasks |
|---------|---------|-------|
| **Purpose** | Develop and test Recipes | Execute deterministic tasks |
| **Session Display** | Current development session | All task list |
| **Use Cases** | Explore webpages, debug Recipes, quick experiments | Run existing Recipes, execute standardized tasks |

**Usage Recommendations**:

- **Use Console for development**: When you want AI to explore how to complete a task (e.g., "help me figure out how to extract data from this webpage")
- **Use Tasks for execution**: When you want to run an existing Recipe or execute tasks that don't need mid-course adjustments

**Important Note**:
âš ï¸ Console auto-executes all operations (file modifications, command execution, etc.). Be careful when using it on sensitive projects. For full manual control, use `claude -p "your task"` from the command line.

## Q: What does Recipe mean?

**A**: A Recipe is a standardized automation task defined by frago, including complete task descriptions, parameter definitions, and execution logic. Think of it as a "saved cooking recipe."

**Why is it called Recipe?**

Just like cooking recipes:
- ğŸ“ **Has clear steps**: Recipe defines the specific workflow to complete a task
- ğŸ”§ **Can adjust parameters**: Like adjusting salt in cooking, Recipes can take different URLs, keywords, etc.
- â™»ï¸ **Can be reused**: Create once, use anytime with the same method

**Recipe Types**:

1. **Atomic Recipe**
   - Solves a single problem
   - Example: `screenshot_tab` - Screenshot current webpage
   - Characteristics: Simple, composable

2. **Workflow Recipe**
   - Chains multiple steps
   - Example: Scrape webpage â†’ Extract data â†’ Save to file
   - Characteristics: Complex, highly automated

**Where to find Recipes?**

- **Local Recipes**: Your local recipes (including ones you created and installed from community)
- **Community Recipes**: Official frago and community-contributed recipe library

**How to use a Recipe?**

1. Go to Recipes page
2. Find the Recipe you need
3. Click "Run" button
4. Fill in required parameters (if any)
5. Execute and view results

**Example**:
```
Recipe: web_extract_to_json
Parameters:
  - url: https://example.com/products
  - selectors: {"title": "h1", "price": ".price"}
  - output_file: products.json

After execution, it will automatically visit the webpage, extract data, and save as JSON
```

## Q: What's the difference between Skills and Recipes?

**A**: Recipes are executable standardized recipes (code), while Skills are methodology documents that guide Claude's work (text). Beginners can ignore Skills initially.

**Simple Analogy**:

- **Recipe**: Standardized cooking steps that machines can execute directly
- **Skill**: Cooking tips and insights that teach chefs how to think

**Detailed Comparison**:

| Feature | Recipe | Skill |
|---------|--------|-------|
| **Nature** | Standardized recipe definition | Markdown document |
| **Execution** | Can run directly | Not executable, read-only |
| **Purpose** | Automate repetitive tasks | Guide AI's decision-making process |
| **Example** | screenshot_tab.py | frago-chrome-navigation.md |
| **Users** | All users | Advanced users |

**When do you need Skills?**

When using Claude Code CLI, Skill documents are automatically loaded to help Claude understand frago's best practices. For example:
- `frago-chrome` skill teaches Claude how to operate Chrome
- `frago-recipe` skill teaches Claude how to write Recipes

**Beginner Advice**:
ğŸ’¡ When starting with frago, focus on using Recipes. Skills are for Claude to read, not for humans to use.

## Q: Why are some Recipes in "Local" and others in "Community"?

**A**: Local shows Recipes already installed on your machine, while Community shows Recipes available for installation from the official repository.

**Three-level Priority**:

frago searches for Recipes in this order:

1. **Project level** (`~/.frago/projects/[run_id]/recipes/`)
   - Recipes specific to current Run project
   - Highest priority

2. **User level** (`~/.frago/recipes/`)
   - Recipes you created or installed
   - Shared across projects

3. **Example level** (Built into frago installation)
   - Official example Recipes
   - For learning and reference

**Local Recipes include**:
- User-level Recipes (ones you installed)
- Recipes installed from Community
- Recipes you created yourself

**Community Recipes**:
- Official frago recipe library
- Hosted on GitHub
- Clicking "Install" will install to User level

**Example**:
```
You see: Local (5) | Community (15)

Local's 5 = 3 you previously installed + 2 you wrote yourself
Community's 15 = Available recipes in official library (excluding installed ones)
```

**Tip**:
ğŸ’¡ Installing Community Recipes requires GitHub CLI configuration, or you'll hit GitHub API rate limits.

## Q: What do the Dashboard charts mean?

**A**: Dashboard shows frago activity statistics for the last 12 hours, including session count, tool call count, and execution status distribution.

**Chart Interpretation**:

**1. System Overview Cards**:
- **Server**: Server status (Online/Offline)
- **Uptime**: Server running time
- **Tasks/Recipes/Skills**: Resource counts

**2. Activity Statistics**:
- **Sessions**: Number of tasks/sessions started in last 12 hours
- **Tool Calls**: Number of operations executed by AI (file reads, commands, etc.)
- **Completed/Running/Errors**: Task status distribution
  - âœ… Completed: Successfully finished
  - ğŸ”„ Running: Currently executing
  - âŒ Errors: Failed execution

**What is a Tool Call?**

Each operation AI performs counts as one Tool Call, for example:
- Read a file
- Execute a shell command
- Screenshot a webpage
- Click a button

One task usually contains multiple Tool Calls.

**How to interpret the data?**

```
Sessions: 5       â†’ You ran 5 tasks today
Tool Calls: 127   â†’ AI executed 127 operations total
Completed: 4      â†’ 4 tasks succeeded
Running: 1        â†’ 1 task still executing
Errors: 0         â†’ No failed tasks
```

**Tip**:
ğŸ’¡ If Tool Call count is very high without corresponding results, the task description might not be clear enough, causing AI to repeatedly explore.
