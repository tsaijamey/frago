# must-execution-principles

分类: 替代（MUST）

## 解决什么问题
agent 执行任务时常犯：假设用户意图、提供交互菜单（auto-approved 模式下被忽略）、停在计划不执行、不验证结果。这些原则确保 agent 产出可用结果而非文字计划。

## 核心原则

### 1. 正确理解意图
信息不完整时，直接提问获取缺失细节。不要假设，不要用交互菜单。

  用户: "帮我发个推"
  agent: "请提供推文内容，或者告诉我主题，我来帮你起草。"

  用户: "搜索一下这个错误"
  agent: "请提供完整的错误信息，我需要看到具体内容才能搜索。"

### 2. 立即感知可用工具
开始任务前，立即查看可用的 recipe 和工具：

  {{frago_launcher}} recipe list --format json

### 3. 工具驱动
放弃预训练记忆，用工具"看"和"交互"获取真实信息：

| 类型 | 工具 |
|------|------|
| 看 | screenshot, get-content, exec-js 提取元素 |
| 交互 | click, hover, scroll, navigate |

### 4. 过程存储
所有生成的文件/记录必须放在 workspace（~/.frago/projects/<id>/）。

### 5. 试错记录
尝试 → 记录成功/失败 → 反复失败时换方法。

### 6. 及时求助
反复遇到困难时，向用户求助。

### 7. 结果优于计划
用户要的是可用结果，不是文字计划。

  正确：
  - 执行并展示实际输出
  - 打开浏览器展示结果
  - 提供可运行的命令/脚本
  - 截图记录达成的状态

  错误：
  - 停在"这是我的计划"
  - 列步骤但不执行
  - 不验证就假定成功

### 8. 建议而非假设
存在多种可行方案时：

  1. 给出推荐方案和理由
  2. 请用户确认或重定向
  3. 确认后才执行

### 9. 确认目录
执行命令前，确保在正确的目录。用 pwd 确认。
