# 使用技巧

## Q: 我想让AI帮我提取网页数据，怎么说？

**A**: 用清晰具体的描述，包含三要素：目标URL、要提取的内容、输出格式。越具体，AI执行越准确。

**三要素公式**：

```
从 [URL] 提取 [具体内容]，保存为 [格式]
```

**示例对比**：

### ❌ 不好的描述（模糊）

```
"帮我抓取这个网站的数据"
→ AI不知道要抓什么，可能提取无关内容
```

### ✅ 好的描述（清晰）

```
"从 https://example.com/products 提取所有商品的名称、价格和库存，
保存为CSV文件，列名为：product_name, price, stock"

→ AI知道：
  - 目标URL
  - 要提取什么字段
  - 输出格式要求
```

**更多示例**：

**示例1: 提取文章列表**
```
从 https://news.example.com 的首页提取前20篇文章的标题和链接，
保存为JSON数组，格式：[{"title": "...", "url": "..."}]
```

**示例2: 分页数据**
```
从 https://shop.example.com/category/electronics 提取前3页的商品数据，
包括：商品名、价格、评分、图片URL，保存为products.json
```

**示例3: 表格数据**
```
从 https://stats.example.com/table 提取页面上的数据表格（id="data-table"），
保存为Excel文件，保留原有格式
```

**高级技巧**：

**1. 指定选择器（可选）**
```
从首页提取用户评论，CSS选择器：.review-card
```

**2. 数据清洗要求**
```
提取价格数据，去除货币符号，转换为数字
```

**3. 翻页处理**
```
从列表页提取数据，点击"下一页"按钮继续，直到没有更多数据
```

**4. 登录后提取**
```
先登录 https://example.com（用户名：xxx，密码从环境变量读取），
然后从个人中心提取订单历史
```

**常见错误与修正**：

| 错误描述 | 问题 | 改进 |
|----------|------|------|
| "抓取这个网站" | 没说抓什么 | "提取所有文章标题和发布时间" |
| "获取价格" | URL不明确 | "从 https://... 页面获取价格" |
| "保存数据" | 格式不明确 | "保存为JSON/CSV/Excel" |
| "提取内容" | 字段不明确 | "提取：标题、作者、日期" |

**提示与注意事项**：

💡 **分步骤描述**：
复杂任务可以分步说明：
```
1. 访问 https://example.com
2. 在搜索框输入"手机"
3. 等待结果加载
4. 提取前10个商品的标题和价格
5. 保存为mobile_prices.json
```

⚠️ **反爬虫注意**：
- 某些网站有反爬虫机制
- 如遇到验证码，AI会提示你
- 遵守网站的robots.txt规则

⚠️ **数据量控制**：
- 首次尝试先提取少量数据（如10条）
- 验证正确后再扩大范围

**完整示例**：

```
任务描述:
从豆瓣电影Top250（https://movie.douban.com/top250）
提取前50部电影的以下信息：
- 电影名称（中文）
- 评分
- 评价人数
- 导演
- 主演（前3位）
- 上映年份

保存为JSON文件，命名为douban_top50.json
格式示例：
[
  {
    "title": "肖申克的救赎",
    "rating": 9.7,
    "votes": 2800000,
    "director": "弗兰克·德拉邦特",
    "actors": ["蒂姆·罗宾斯", "摩根·弗里曼", "鲍勃·冈顿"],
    "year": 1994
  }
]

注意：
- 需要翻页到第2页（每页25条）
- 提取完成后显示总数量确认
```
